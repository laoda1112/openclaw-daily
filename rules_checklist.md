# 发布文章规则清单（可复用）

## 内容规范
- 标题清晰，包含核心关键词（OpenClaw / 版本号 / 功能名）
- 摘要 2–4 句，覆盖读者收益与适用场景
- 正文按“概述 → 步骤 → 注意事项 → 结论”结构
- 术语前后一致（如“网关”“任务”“子任务”）
- 所有命令、路径、参数与实际一致

## 格式规范
- 文档使用 Markdown
- 代码块必须标注语言（bash / yaml / json）
- 文件命名使用 snake_case
- 小标题不超过两级（## / ###）

## 可发布标准
- 模板可复用、结构清晰
- 示例文章符合模板结构
- 可独立阅读，不依赖内部联系人解释
- 不包含敏感信息（API Key、私有地址、内网 IP 等）

## 质量检查
- 语句通顺、无明显语病
- 关键命令可执行（不含错误参数）
- 步骤数量适中（3–7 步）
- 术语首次出现有简短解释

## 发布前自检
- [ ] 标题与摘要对齐
- [ ] 版本号/日期已确认
- [ ] 重要参数加粗或代码块突出
- [ ] 参考链接可访问
