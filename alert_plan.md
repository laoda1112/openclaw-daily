# 告警方案（轻量版）

## 目标
- 站点宕机/内容过期/发布失败能在 5–10 分钟内被发现
- 告警可读、可追踪、可降噪

## 告警触发条件
1) **可用性失败**
- 关键 URL 非 200 或响应超时
- 连续失败 2 次触发（降低抖动）

2) **内容生成异常**
- sitemap/feed 最新时间 > 48 小时
- 首页关键标识文本缺失

3) **发布链路失败**
- GitHub Actions / Cloudflare Pages 最新一次部署状态非 success

## 告警通道
- **主通道**：IM Webhook（Slack/飞书/企业微信）
- **备份**：邮件（可接入 webhook 转发）

## 告警内容模板
```
【发布站监控告警】
站点：<SITE_URL>
时间：<UTC 时间>
问题：
- <失败项1>
- <失败项2>
最近部署：<平台/状态/时间>
```

## 降噪策略
- 连续 2 次失败才告警
- 成功恢复后发送“恢复通知”
- 业务低峰期允许更高阈值（如内容过期 72h 才告警）

## 运行建议
- 脚本通过 cron 或 CI 定时执行（每 5–10 分钟）
- 监控脚本使用环境变量配置站点与 webhook
- 配合 CI “部署完成后立即执行一次监控”

## 责任与处置
- 监控负责人：运营/开发 On-call
- 处置优先级：可用性 > 发布链路 > 内容异常
