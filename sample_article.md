# 3 分钟用 OpenClaw 自动发布你的公告

## 摘要
如果你需要在多个渠道快速同步公告，这篇文章教你用 OpenClaw 的自动发布脚本完成一次“一键发布”。你会得到一套可复用流程，并能在 3 分钟内跑通。适用于日常运营公告与版本发布同步。

## 背景 / 痛点
手动复制粘贴容易遗漏、格式不统一，也难以追踪发布结果。通过脚本化的发布流程，可以保证内容一致、节省时间。

## 方案概述
- 准备统一的公告内容（Markdown）
- 配置发布目标渠道
- 运行发布脚本并检查结果

## 详细步骤
### 步骤 1：准备公告内容
创建 `release_note.md`，确保标题与摘要完整。

```markdown
# OpenClaw 0.3.2 发布说明

## 摘要
新增网关监控与失败重试；优化任务日志查询性能。
```

### 步骤 2：配置发布目标
在配置文件中写入要发布的渠道列表：

```yaml
channels:
  - telegram
  - discord
  - slack
```

### 步骤 3：运行自动发布脚本
在项目根目录执行：

```bash
openclaw publish --file release_note.md --config publish.yaml
```

执行完成后，终端会返回发布结果摘要。

## 注意事项 / 常见问题
- 发布失败时先检查网络与权限
- 渠道配置需要与实际机器人权限一致
- 同步到多个渠道时，注意频控

## 结论 / 下一步
通过自动发布脚本，你可以把公告从“手动分发”变成“标准流程”。接下来建议将脚本加入日常发布清单，保证每次发布可追踪、可复用。
